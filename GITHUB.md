## Releases & version control

- Tags: `v<semver>` (e.g. `v0.1.1`), created on `main`.
- Each GitHub Release carries three assets: `Flux_<ver>+jammy_amd64.deb`, `+noble`, `+resolute`.
- Version is suffixed per Ubuntu codename (`0.1.1+jammy`) because the three builds
  link against different webkit2gtk ABIs and share one apt pool.

### Cutting a release (one command)

```bash
sg docker -c "packaging/release.sh 0.1.2"
```

Does, in order: semver + clean-tree + tag checks, bump version in `package.json`
/ `tauri.conf.json` / `Cargo.toml` (+ `Cargo.lock`), commit `release: v0.1.2`,
tag, build .debs for all three codenames in Docker, refresh the signed apt repo,
push `main` + tag, create the GitHub Release with the .debs, deploy the apt repo
to Pages.

### Manual pieces (what release.sh automates)

```bash
bash packaging/build.sh [codename]        # .debs into packaging/dist/<codename>/
bash packaging/apt-repo/build.sh          # reprepro: publish/ tree, signed
bash packaging/apt-repo/deploy.sh         # force-push publish/ to flux-apt
gh release create v0.1.2 packaging/dist/*/Flux_0.1.2+*_amd64.deb -t "Flux v0.1.2" -n "notes"
```

Docker commands need `sg docker -c "..."` if the shell session predates docker
group membership.

## apt repository

- Layout: standard `dists/<codename>/` + `pool/`, generated by reprepro,
  one distribution per supported Ubuntu release: `jammy` (22.04), `noble`
  (24.04), `resolute` (26.04). Ubuntu 20.04 unsupported (no webkit2gtk-4.1).
- Serves only the newest version per codename (reprepro default); older .debs
  remain downloadable from GitHub Releases.
- Pages config: `flux-apt` repo, branch `main`, path `/`, `.nojekyll` present.

### User install

```bash
sudo install -d -m 0755 /etc/apt/keyrings
curl -fsSL https://ydvsahil03.github.io/flux-apt/pubkey.gpg | sudo gpg --dearmor --yes -o /etc/apt/keyrings/flux.gpg
echo "deb [signed-by=/etc/apt/keyrings/flux.gpg] https://ydvsahil03.github.io/flux-apt $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/flux.list
sudo apt update && sudo apt install flux
```

Updates arrive via normal `apt upgrade`.

## Troubleshooting

- **apt 404 / "does not have a Release file"**: Pages not deployed or still
  building — check https://ydvsahil03.github.io/flux-apt/pubkey.gpg returns 200.
- **"no valid OpenPGP data"** during install: the curl above 404'd and wrote a
  garbage keyring; re-run the key step once Pages is live.
- **reprepro "already has" warnings** on rebuild: harmless — same version
  re-included is skipped.
- **Stale .debs in packaging/dist/**: fixed in build.sh (clears the work
  volume's bundle dir first); if old `Vantage_*`/unsuffixed files appear,
  delete them before running apt-repo/build.sh.
